# Вимоги до оформлення Prolog-коду для «Задача» / «Гра»

## 0) Критично важливо
- **Попередження Singleton variables — це помилка.**
  - Якщо змінна не використовується, замінити на `_`.
  - Якщо змінна потрібна для логіки, вона має реально брати участь у виразі/обмеженні/виклику.
- Імена файлів — **лише латиницею**, з транслітерацією прізвища.

## 1) Документація перед кожним предикатом (обов’язково)
Перед визначенням **кожного** предиката має бути коментарний блок:
- **mode-індикатори параметрів** (++, +, --, -, ?),
- **Purpose** (коротко: що робить предикат),
- **аналіз мультипризначенності**: перелік усіх змістовних режимів з коротким описом,
- **незмістовні режими**: коротко пояснити, чому їх не підтримано,
- блок `/** <examples> ... */` з 2–4 прикладами запитів у стилі SWI/SWISH та коротким описом очікуваного результату.

### 1.1 Позначення mode-індикаторів
- `++` — обов’язково конкретизований (вхідний)
- `+`  — конкретизований або частково конкретизований
- `--` — обов’язково вільна змінна (вихідний)
- `-`  — вільна змінна або конкретизований «для перевірки»
- `?`  — універсальний: вхідний або вихідний

## 2) Мультипризначенність
- Для кожного предиката описуються **всі змістовні призначення** з mode-індикаторами.
- Якщо змістовних призначень мало — це нормально, але треба **пояснити відсутність інших**.
- Усі можливі комбінації аналізуються теоретично, але у коментарях описуються лише змістовні.

## 3) Коментарі по вбудованих предикатах
- Неочевидні built-ins і функтори треба стисло коментувати (приклади):
  - `findall/3`, `setof/3`, `bagof/3`
  - `permutation/2`
  - `maplist/2`
  - `labeling/2`, `element/3`, `circuit/1` (CLP(FD))
  - `statistics/2`, `time/1` (вимірювання)
- Очевидні (типу `is`, `=`, `append`, `+`, `div`) можна не коментувати.

## 4) Відступи (“двовимірний синтаксис”)
Тіло предикатів оформлюється з обов’язковими відступами та вирівнюванням.
Приклад стилю:
```prolog
foo(X, Y, Z) :-
    sort(X, X1),
    Z is X1 + 1,
    insert(Z, Y).
````

## 5) Рекомендований шаблон для кожного предиката

Використовувати такий формат:

```prolog
%% predicate_name(++A, +B, -C) is det.
%  Purpose:
%    ...
%
%  Modes (meaningful):
%    1) (++A, +B, -C)  ...
%    2) (++A, -B, -C)  ...
%
%  Modes (not meaningful):
%    (-A, ?, ?)  Не має змісту, оскільки ...
%
%  Notes / built-ins:
%    - findall/3: ...
%
/** <examples>
?- predicate_name(Arg1, Arg2, Out).
% очікується: ...
*/
predicate_name(A, B, C) :-
    ...
.
```

## 6) Перевірка перед здачею (самоконтроль)

* Код завантажується у SWI-Prolog без попереджень про singleton.
* Для кожного предиката є pldoc + examples.
* Стиль відступів однаковий у всіх файлах.
