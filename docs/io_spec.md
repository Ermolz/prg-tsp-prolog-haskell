# Специфікація введення/виведення (I/O) для TSP

## 1) Основний формат інстансу: матриця N*N у файлі *.tsp

### 1.1 Правила парсингу
- Рядки, що починаються з `#`, є коментарями та ігноруються.
- Перша значуща (непорожня і некоментарна) строка: `N` (ціле число).
- Далі йдуть `N` рядків по `N` цілих чисел, розділених пробілами або табуляцією.
- Діагональ матриці бажано `0`.
- Відстані мають бути не від’ємніми цілими числами.
- Підтримується як симетрична, так і асиметрична матриця (не робити припущення про симетрію).

### 1.2 Приклад файлу `data/instances/n5_demo.tsp`
```text
# tsp-matrix v1
5
0 2 9 10 7
2 0 6 4 3
9 6 0 8 5
10 4 8 0 1
7 3 5 1 0
```

## 2) Угода про індексацію міст

* Міста індексуються як `1..N`.
* Тур у виведенні є **замкнутим циклом**, тобто перше місто повторюється наприкінці.

## 3) Формат результату (однаковий для Prolog і Haskell)

Один рядок:

* `cost=<ціле>` пробіл `tour=[1,...,1]`

### 3.1 Приклад

```text
cost=17 tour=[1,2,4,5,3,1]
```

## 4) Вимоги до узгодженості між мовами

* Prolog і Haskell зчитують той самий файл `*.tsp`.
* Обидві реалізації друкують результат у точно такому ж форматі, як описано вище.
* Якщо існують кілька оптимальних турів, допускається різний `tour`, але `cost` має збігатися.
